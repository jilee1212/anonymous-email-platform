tasklist.md: 영구 익명 이메일 플랫폼 구축 과정
1. 개요
이 TaskList는 '영구 익명 이메일 플랫폼'의 MVP를 구축하기 위한 상세 작업 목록입니다.

2. 상세 Task List
Phase 1: 기획 및 설계 (Planning & Design)
[ ] '이메일 주소 + 접근 키' 기반 모델 상세 설계:

이메일 주소는 [무작위 문자열]@[도메인] 형태로 생성합니다.

'접근 키'는 사용자가 쉽게 기억하기 어려운 12개의 단어 조합(Wordlist) 또는 64자 무작위 문자열로 정의합니다.

[ ] UI/UX 설계:

메인 페이지에 '새 이메일 생성' 버튼과 '기존 이메일 접속' 폼을 명확히 구분하여 배치합니다.

접근 키 생성 결과 화면에는 키를 강조하고, 복사 버튼 및 "키를 잃어버리면 복구할 수 없습니다."와 같은 강력한 경고 문구를 포함합니다.

[ ] DB 스키마 설계:

users 테이블: email_address (string, unique), access_key (string, encrypted)

emails 테이블: id (int, primary key), user_email (string, foreign key), sender (string), subject (string), body (text), received_at (datetime)

Phase 2: 인프라 구축 (Infrastructure Setup)
[ ] 도메인 및 서버 구매:

[자유로운 도메인명]을 구매하고, Linux 기반의 클라우드 서버(AWS EC2, GCP Compute Engine 등)를 구성합니다.

[ ] DNS 레코드 설정:

도메인 관리 페이지에서 A, MX 레코드를 설정하여 이메일 수신을 위한 준비를 합니다.

스팸 방지를 위해 SPF, DKIM, DMARC 레코드를 추가합니다.

[ ] 웹 서버 및 DB 설치:

Nginx 또는 Apache를 설치하고, PostgreSQL 또는 MongoDB를 설치합니다.

Phase 3: 서버 개발 (Server-Side Development)
[ ] 메일 서버 설정:

Postfix 또는 Exim과 같은 메일 서버 소프트웨어를 설치하고, main.cf 파일에 'Catch-all' 주소(예: @yourdomain.com)로 들어오는 모든 이메일을 특정 스크립트로 전달하도록 설정합니다.

[ ] 메일 파싱 및 DB 저장 스크립트 개발:

수신된 메일 데이터를 파싱(발신자, 제목, 본문 등 추출)하여 DB에 저장하는 백그라운드 스크립트를 작성합니다.

[ ] API 개발:

POST /api/generate-email: 무작위 이메일 주소와 암호화된 접근 키를 생성하고 DB에 저장 후, 이메일 주소와 평문 접근 키를 반환하는 API를 개발합니다.

POST /api/check-access: 사용자가 입력한 이메일 주소와 접근 키를 검증하고, 일치하면 해당 계정의 받은편지함 데이터를 반환하는 API를 개발합니다.

GET /api/inbox/:email: (선택 사항) 인증이 성공한 후, 특정 이메일 주소의 받은편지함 데이터를 조회하는 API를 개발합니다.

[ ] 어뷰징 방지 적용:

express-rate-limit 미들웨어를 사용하여 IP 기반으로 이메일 생성 요청 횟수를 제한합니다.

Phase 4: 프론트엔드 개발 (Frontend Development)
[ ] 메인 페이지 개발:

HTML, CSS(Tailwind CSS 또는 Bootstrap 사용)로 UI를 구현합니다.

'새 이메일 생성' 버튼 클릭 시 POST /api/generate-email을 호출하는 JavaScript 로직을 작성합니다.

[ ] 이메일 및 키 결과 페이지 개발:

API 응답을 받아 이메일 주소와 접근 키를 보여주는 페이지를 만듭니다.

클립보드 복사 기능을 추가하고, "이 키를 잃어버리면 복구할 수 없습니다." 경고 문구를 눈에 띄게 표시합니다.

[ ] 받은편지함 접속 페이지 개발:

이메일 주소와 접근 키를 입력받는 폼을 개발합니다.

폼 제출 시 POST /api/check-access를 호출하고, 성공 시 받은편지함 페이지로 이동합니다.

[ ] 받은편지함 페이지 개발:

받은 메일 목록을 보여주는 테이블 또는 리스트를 구현합니다.

각 메일 항목 클릭 시 상세 내용을 보여주는 모달 또는 새로운 페이지를 만듭니다.

Phase 5: 테스트 및 배포 (Testing & Deployment)
[ ] 핵심 시나리오 테스트:

생성: 새 이메일 주소와 키를 성공적으로 생성하고, 복사합니다.

재접속: 브라우저를 종료한 후 다시 접속하여, 복사해 둔 키와 이메일 주소로 로그인합니다.

수신: 외부 이메일 계정(Gmail 등)에서 방금 생성한 이메일 주소로 테스트 메일을 보내고, 받은편지함에서 수신을 확인합니다.

[ ] 에러 핸들링 테스트:

잘못된 접근 키를 입력했을 때, 서버가 올바른 에러 메시지를 반환하고 프론트엔드가 이를 표시하는지 확인합니다.

[ ] 보안 점검:

SQL Injection, XSS 등 기본적인 웹 취약점 테스트를 수행합니다.

[ ] 프로덕션 배포 및 모니터링:

완성된 플랫폼을 프로덕션 서버에 배포합니다.

Grafana, Prometheus 등 모니터링 도구를 설치하여 서버 상태 및 이메일 수신/발송 현황을 지속적으로 모니터링합니다.

이 문서는 프로젝트의 전체적인 로드맵을 제공하며, 각 단계별로 구체적인 작업을 제시하고 있습니다. 혹시 특정 단계에 대해 더 깊이 있는 논의가 필요하거나, 다른 기술 스택을 고려하고 싶으시다면 언제든지 말씀해주세요.
